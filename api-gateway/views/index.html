الأطروحة العلمية وخطة العمل

الملخص العلمي Abstract

في النظم البرمجية القائمة على الخدمات المصغرة (Microservices)، تعد بوابة واجهة برمجة التطبيقات (API Gateway) نقطة الدخول الحيوية الوحيدة التي تدير حركة المرور المتجهة من الشمال إلى الجنوب (North/South Traffic) بين العملاء والخدمات الخلفية. نظراً لموقعها المركزي، فإن كفاءة خوارزميات توجيه الطلبات تعد عاملاً حاسماً في الأداء العام للنظام وتجنب الاختناقات الهيكلية.

يستقصي هذا البحث الأسس الخوارزمية لاستراتيجيات توزيع الأحمال الأساسية، وتحديداً: Round Robin، Weighted Least Connections (WLC)، وHash-Based Routing. الهدف الرئيسي هو سد الفجوة بين التحليل الخوارزمي النظري البحت —باستخدام ترميز Big-O— وبين الأداء التجريبي الفعلي لهذه الخوارزميات عند نشرها كبرمجيات وسيطة نشطة.

من خلال التقييم الصارم لهذه الاستراتيجيات تحت أنماط حركة مرور متنوعة وديناميكية، تحدد الدراسة الآليات المثلى لتقليل زمن الاستجابة (Latency) وزيادة الإنتاجية (Throughput). في النهاية، تقدم النتائج دليلاً عملياً حاسماً لمهندسي النظم لضبط تكوينات بوابات الـ API، وتجاوز قيود التوجيه الساكن، وتلبية متطلبات الأداء العالي للبيئات القابلة للتوسع بشكل كبير.

المشكلة الخوارزمية المحددة Specific Algorithmic Problem

نوع المشكلة: Comparative Study & Algorithmic Adaptation
محور التحليل: Time & Space Complexity (Centralized vs Decentralized)

يركز البحث على تعقيد الوقت والمساحة (Time & Space Complexity) لخوارزميات التوجيه على مستوى التطبيق بين النهج المركزي والنهج اللامركزي. تحديداً، يحلل البحث العبء الخوارزمي الإضافي (Overhead) لصيانة جداول التوجيه وحالات اكتشاف الخدمات في الوقت الفعلي (Space Complexity) مقابل وقت التنفيذ والتأخير الناتج عن اختيار مثيل الخدمة الأمثل (Time Complexity).

🏢 السيناريو المركزي (API Gateway)

يسبب اختناقاً معمارياً مركزياً نتيجة فرض قفزة شبكية إضافية (Extra Network Hop) وتراكم الطلبات في طابور واحد، لكنه يتميز بتعقيد مساحة أقل O(n) حيث يتم الاحتفاظ بحالة الجدول في مكان واحد.

🕸️ السيناريو الموزع (Service Mesh - Sidecar)

يوزع عبء التوجيه محلياً دون قفزات شبكية إضافية، ولكنه يعتمد على تكرار الحالة (State Replication) مما يرفع تعقيد المساحة إلى O(k × n)، بالإضافة إلى عبء المزامنة الشبكية (Gossip Traffic) وتأخير الجمود (Stale Data / Herd Effect).

تقوم الدراسة بمقارنة كمية لكيفية تدرج عبء المزامنة ومنطق الاختيار مع النمو الأسي لعدد مثيلات الخدمات المصغرة (n) وحجم الطلبات المتزامنة (m).

خطة العمل المقترحة Proposed Research Plan

1

الأسبوع الأول: النمذجة النظرية ومراجعة الأدبيات

تطبيق مفاهيم "تحليل الخوارزميات" لنمذجة أوقات التنفيذ في الحالات الفضلى والمتوسطة والأسوأ لمنطق التوجيه رياضياً باستخدام نماذج شبكات الانتظار (Queuing Networks - QN). صياغة الفروق المعمارية بين التجميع المركزي واللامركزي.

2

الأسبوع الثاني: إعداد بيئة المحاكاة المحلية

تطوير ونشر محاكاة خاضعة للرقابة باستخدام الحاويات الخفيفة (Docker Compose) بدلاً من منصات الأوركسترا المعقدة. تشغيل بوابة API أساسية متصلة بخدمات مصغرة خلفية لقياس تخصيص الموارد وسلوك الخوارزميات بدون أعباء بنية تحتية ثقيلة.

3

الأسبوع الثالث: جمع البيانات واختبار الجهد

استخدام أدوات توليد الأحمال والجهد (مثل Artillery / JMeter) لقياس زمن الاستجابة الفعلي واستهلاك الذاكرة والـ CPU تحت أحمال متزايدة ومتنوعة (مثال: طلبات كثيفة القراءة مقابل عمليات مكثفة حاسوبياً). جمع البيانات لمطابقتها مع نماذج Big-O النظرية.

4

الأسبوع الرابع: صياغة التقرير النهائي والاستنتاجات

دمج النماذج الرياضية النظرية بالنتائج التجريبية المستخلصة من بيئة المحاكاة لتوضيح كيف تملي تعقيدات الوقت والمساحة اختيار خوارزمية التوجيه المناسبة حسب النطاق (Scale)، وصياغة مبادئ توجيهية لمهندسي البرمجيات.

أوراق علمية ومراجع ذات صلة Related Papers & References

Performance Modeling and Analysis of Design Patterns for Microservice Systems

Riccardo Pinciroli, Aldeida Aleti, Catia Trubiani (2023)

تستخدم الدراسة "شبكات الانتظار" لنمذجة الأداء وتحليل تعقيد الوقت لأنماط مثل بوابة الـ API (التجميع والتفويض)، وتوضح كيف تسبب أعباء العمل المتنوعة انتقال نقطة الاختناق (Bottleneck Switch) بين الخدمات.

Kubernetes Networking: Comparative Insights into API Gateways and Service Mesh Implementations

Sami Kuikka (2024)

تقارن الأطروحة بين بوابات الـ API المركزية وشبكات الخدمات الموزعة من حيث تعقيد المساحة واستهلاك الموارد، وتناقش استخدام تقنيات (eBPF/Cilium) كبديل لنمط Sidecar لتقليل استهلاك الذاكرة.

AI-Powered API Management: Intelligent traffic routing and adaptive scaling

Prasanna Kumar Natta (2025)

تبحث الورقة في كيفية استخدام الخوارزميات الديناميكية المعززة بالذكاء الاصطناعي لتحسين توجيه الطلبات في الوقت الفعلي، وتثبت خفض زمن التأخير بنسبة 24.7% مقارنة بالخوارزميات الساكنة.

API Gateways and Microservice Architectures

Richard Trebichavský (2021)

توفر مرجعاً عملياً لإعداد بيئة تجريبية باستخدام أدوات فحص الجهد لقياس التأخير واستهلاك الموارد في بوابات (Kong, KrakenD)، وتقييم تكلفة القفزة الشبكية الإضافية للبوابات المركزية.
